Para nosso trabalho de desenvolver uma "Pokedex" para diversas gerações de pokemons, pegamos por base a API que o professor disponibilizou no link do próprio trabalho.


A partir disso buscamos definir, quais seriam os elementos principais que iriam ser utilizados, dentre inúmeras opções possíveis, indo desde a raridade do pokemon, seus locais de encontro, sua altura e peso, e até mesmo a versão desejada do pokemon em questão.


Dentre os dados que achamos mais relevantes exibir, estão:
  • Imagem
  • Estatísticas
  • Elemento(s)
  • Forma "Shiny"


    • O principal metódo utilizado, foi o de trazer os dados de cada pokemon, a partir disso que começamos a desenvolver quais dados iríamos exibir. Realizamos a integração da caixa de pesquisa para que a mesma fosse utilizada para localizar os pokemons tanto por seu nome quanto por seu número de registro na pokedex. A partir do momento que o método para retornar os dados do pokemon passou a estar funcional, basicamente fizemos o javascript percorrer os campos respectivos a cada informação que desejávamos exibir.


    Após exibirmos as informações desejadas, criamos botões responsáveis por exibir os pokemons sucessores ou antecessores, seguindo por parametro o seu número de cadastro. Além disso, criamos um botão para exibir a forma "Shiny" do pokemon, pegando o sprite visual dela tendo por base a forma padrão da geração escolhida, assim pode-se alternar entre a forma padrão e shiny sem ocorrer a quebra de alinhamento.
